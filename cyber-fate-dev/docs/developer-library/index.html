Architecture Audit

审查结论：当前项目已经是前后端分离运行形态

PASS Next frontend 不再暴露 src/app/api/reports 路由，也不直接导入模型调度、PDF 生成或报告存储模块。

前端通过 src/lib/frontend/backendApi.ts 生成 backend API 地址；后端入口为 src/backend/server.ts，独立负责报告生成、读取、PDF 与模型错误返回。

运行时结构

层目录职责禁止事项
Frontend src/app, src/components, src/lib/frontend 页面路由、React 组件、浏览器 API base URL、报告展示。 不能 import src/lib/llm, src/lib/agents, src/lib/pdf, reportStore。
Backend src/backend, src/lib/agents, src/lib/llm, src/lib/pdf, src/lib/report HTTP API、模型调用、Zod 校验、文件存储、PDF 输出。 不能 import Next 页面、React 组件或依赖浏览器运行时。
Shared Contract src/lib/schemas, src/lib/report/reportSchema.ts 前后端共享类型与 Zod schema。 不要放业务副作用、文件 I/O 或模型调用。

前端样式修改指南

修改目标推荐文件说明
全局字体、背景、主题色src/app/globals.css, tailwind.config.ts优先改 token，不要散落硬编码颜色。
首页视觉src/app/page.tsx只负责入口和跳转，不应调用 backend。
访谈表单src/app/intake/IntakeForm.tsx字段变更必须同步 src/lib/schemas/intake.ts。
生成进度和错误 toastsrc/app/generate/GenerateClient.tsx只能通过 backendApiUrl() 调 API；错误结构来自 backend。
报告预览和印章src/components/report/*适合视觉迭代。数据只来自 CyberFateReport schema。

样式改造的成熟路径是：先修改共享 token，再调整组件布局，最后用浏览器验证桌面和移动端。避免为了样式读取文件系统或调用模型接口。

后端 API 参考

方法路径用途主要响应
GET/health健康检查与当前模型模式。{ ok, service, mode, timestamp }
POST/api/reports校验 intake，调用模型 pipeline，保存报告。{ id, mode, report, artifacts }
GET/api/reports/:id读取报告 JSON。CyberFateReport 或 REPORT_NOT_FOUND
GET/api/reports/:id/pdf生成并下载 PDF。application/pdf 或 PDF_GENERATION_FAILED

开发交接摘要

  • 做前端样式时，只改 src/app、src/components、Tailwind/global CSS 和 schema 类型。
  • 做模型、PDF、存储、API 错误时，只改 backend 和 src/lib 服务模块。
  • 新增 API 必须先写在 backend，再让 frontend 通过 backendApiUrl() 访问。
  • 不要恢复 Next src/app/api 路由，避免回到全栈单体耦合。
  • 真实 API key 只写入本地 .env 或 .env.local，不要提交。
