# Cyber Fate 当前主控 Prompt

## 当前产品目标

Cyber Fate / 赛博天命局必须是一个可本地运行、可演示、可生成 PDF 的赛博玄学白皮书产品。

## 当前硬性约束

1. 用户报告必须由模型生成。
2. 不允许使用本地替代报告作为生成结果。
3. 缺少 key、代理失败、模型输出不合规时，接口必须返回结构化错误。
4. 生成页必须在右下角显示错误反馈，并提供可复制 trace 信息。
5. PerceptLeap 是默认模型路径：`CYBER_FATE_LLM_MODE=perceptleap`。
6. 角色调用应尽量并行：访谈与检索并行，视觉导演与审阅并行。
7. 模型输出必须通过 Zod schema 后才能保存、预览或导出 PDF。
8. 报告仅供娱乐与自我观察，不作为医学、法律、投资或人生重大决策依据。

## 当前验收

- `pnpm test` 通过。
- `pnpm lint` 通过。
- `pnpm build` 通过。
- `/api/reports` 在模型配置错误时返回结构化错误。
- `/generate` 在右下角显示错误 toast。
- 旧样例报告入口不存在。
- 代码中不存在本地替代报告生成链路。
